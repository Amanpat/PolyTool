# Dev Log Directory

Dev logs are mandatory for every meaningful work unit in PolyTool.

**Actual dev log files live at:** `docs/dev_logs/YYYY-MM-DD_<slug>.md`

This Obsidian directory is a pointer — not a duplicate store. Browse dev logs in the native filesystem view or via the `docs/dev_logs/` path.

---

## Naming Convention

```
docs/dev_logs/YYYY-MM-DD_<slug>.md
```

Examples:
- `docs/dev_logs/2026-03-21_phase1_docs_closeout.md`
- `docs/dev_logs/2026-03-29_gabagool22_crypto_analysis.md`
- `docs/dev_logs/2026-03-29_gabagool_strategy_rebuild.md`

---

## What Belongs in a Dev Log

- Scope: one work packet, one clear objective
- Code changes and the tests run (with exact pass/fail counts)
- Decisions made and why
- Open questions or blockers discovered
- Codex review summary (tier, issues found, issues addressed)
