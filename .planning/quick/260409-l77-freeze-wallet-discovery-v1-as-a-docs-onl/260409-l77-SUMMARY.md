## Decisions Made

1. **Spec naming**: `SPEC-wallet-discovery-v1.md` (descriptive slug, not numbered). Follows `SPEC-wallet-scan-v0.md` and `SPEC-alpha-distill-v0.md` patterns. Numbered specs are reserved for gate/SimTrader/infrastructure specs with ordering relationships.

2. **Human review gate is mandatory**: `scanned -> promoted` is an explicitly invalid transition. The path is `scanned -> reviewed -> promoted` where `review_status = 'approved'` must be set by a human operator. No code path bypasses this.

3. **`watched` state in schema, not operational in v1**: Forward-compatibility field. Loop B (Alchemy WebSocket) is out of scope for v1. The `promoted -> watched` transition must not be implemented until Loop B ships.

4. **Cloud LLM wallet analysis deferred**: Current PLAN_OF_RECORD authorizes Tier 1 free cloud APIs for RIS evaluation scoring only. Wallet analysis extension requires a PLAN_OF_RECORD Section 0 update — a human decision.

5. **Insider scoring deferred**: Architect review (2026-04-09) identified that a single averaged-p0 binomial test is mathematically incorrect for a multi-category prediction market. Per-bucket calibration test design is required before implementation.

6. **Discovery tables in ClickHouse**: Consistent with the one-sentence DB rule (ClickHouse = live streaming writes). DuckDB's role is unchanged.
