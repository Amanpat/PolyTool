**packages/research/scheduling/scheduler.py** — full implementation:

```
"""RIS v1 APScheduler background scheduler.

Registers named periodic jobs for all active RIS ingestion pipelines.
Twitter/X ingestion is explicitly NOT scheduled (deferred — no live
fetcher exists yet; see RIS_02 social ingestion spec).

Job callables are thin wrappers around existing CLI main() functions.
They accept no arguments (APScheduler calls them with no args).

Usage (programmatic):
    from packages.research.scheduling.scheduler import start_research_scheduler
    scheduler = start_research_scheduler()
    # runs in background; call scheduler.shutdown() to stop

Usage (injectable/testable):
    scheduler = start_research_scheduler(
        _scheduler_factory=lambda: FakeScheduler(),
        _job_runner=my_runner,
    )
"""
```

**JOB_REGISTRY** — a list of dicts, each with:
  - `id`: str (e.g. "academic_ingest")
  - `name`: str (e.g. "ArXiv academic ingestion")
  - `trigger_description`: str (human-readable cron, e.g. "every 12h at 06:00 and 18:00")
  - `callable_name`: str (name of the job function in this module)

Register these 8 jobs (per RIS_06 spec, Twitter/X excluded):

| id | name | schedule | job callable |
|----|------|----------|-------------|
| academic_ingest | ArXiv academic ingestion | CronTrigger(hour="6,18") | _job_run_academic_ingestion |
| reddit_polymarket | r/polymarket ingestion | CronTrigger(hour="0,6,12,18") | _job_run_reddit_polymarket |
| reddit_others | Other subreddits ingestion | CronTrigger(hour=3) | _job_run_reddit_others |
| blog_ingest | Blog/RSS ingestion | CronTrigger(hour="2,6,10,14,18,22") | _job_run_blog_ingestion |
| youtube_ingest | YouTube transcript ingestion | CronTrigger(day_of_week="mon", hour=4) | _job_run_youtube_ingestion |
| github_ingest | GitHub README ingestion | CronTrigger(day_of_week="wed", hour=4) | _job_run_github_ingestion |
| freshness_refresh | Freshness tier recalculation | CronTrigger(day_of_week="sun", hour=2) | _job_run_freshness_refresh |
| weekly_digest | Weekly research digest | CronTrigger(day_of_week="sun", hour=8) | _job_run_weekly_digest |

**Job callable implementations** — each is a module-level function:

- `_job_run_academic_ingestion()`: calls `research_acquire.main(["--search", "prediction markets microstructure", "--source-family", "academic", "--no-eval"])` and also `research_acquire.main(["--search", "market microstructure liquidity", "--source-family", "academic", "--no-eval"])`. Import `tools.cli.research_acquire` as `research_acquire` at call time (lazy import inside function body, so tests can monkeypatch without loading APScheduler at module level).

- `_job_run_reddit_polymarket()`: calls `research_acquire.main(["--url", "https://www.reddit.com/r/polymarket/", "--source-family", "reddit", "--no-eval"])`. Note: LiveRedditFetcher.fetch() accepts a subreddit URL and returns posts from that subreddit.

- `_job_run_reddit_others()`: calls `research_acquire.main(["--url", url, "--source-family", "reddit", "--no-eval"])` for each URL in `_REDDIT_OTHERS_URLS = ["https://www.reddit.com/r/PredictionMarkets/", "https://www.reddit.com/r/Kalshi/"]`.

- `_job_run_blog_ingestion()`: calls `research_acquire.main(["--url", url, "--source-family", "blog", "--no-eval"])` for each URL in `_BLOG_FEED_URLS` list (2-3 hardcoded blog URLs relevant to prediction markets).

- `_job_run_youtube_ingestion()`: calls `research_acquire.main(["--search", "prediction markets explained", "--source-family", "academic", "--no-eval"])` — note: YouTube search uses --url not --search; use `--url "https://www.youtube.com/results?search_query=prediction+markets"` with source-family youtube.

- `_job_run_github_ingestion()`: calls `research_acquire.main(["--url", url, "--source-family", "github", "--no-eval"])` for each URL in `_GITHUB_REPO_URLS` (2-3 relevant repos like py-clob-client, polymarket-client).

- `_job_run_freshness_refresh()`: calls `research_acquire.main(["--search", "prediction markets 2026", "--source-family", "academic", "--no-eval"])`. This is a lightweight placeholder that re-scans recent academic content; full freshness recalculation logic will come in a later RIS task.

- `_job_run_weekly_digest()`: imports and calls `from tools.cli.research_report import main as report_main` then `report_main(["digest", "--window", "7"])`.

**_job_runner parameter**: `start_research_scheduler` accepts an optional `_job_runner: Callable[[str], None] | None = None`. When provided, each job's APScheduler callable is replaced with `lambda: _job_runner(job_id)`. This allows full offline testing without APScheduler.

**_scheduler_factory parameter**: `start_research_scheduler` accepts an optional `_scheduler_factory: Callable[[], Any] | None = None`. When provided, uses that factory instead of `BackgroundScheduler()`. When None (default), APScheduler must be installed; raise `ImportError` with clear message if not available.

**APScheduler import**: guard with try/except ImportError at top of `start_research_scheduler()` only (not module level), so `JOB_REGISTRY` and job callables are always importable even without APScheduler installed.

**Return value**: `start_research_scheduler()` returns the scheduler instance (started).

**`run_job(job_id: str) -> int`**: a public function that looks up `job_id` in JOB_REGISTRY, finds the corresponding callable, calls it, and returns 0 on success or 1 on exception (logging the error). This is used by the CLI `run-job` subcommand.

**tools/cli/research_scheduler.py** — three subcommands: `status`, `start`, `run-job`.

```
"""CLI entrypoint for RIS v1 background scheduler management.

Subcommands:
  status     List all registered jobs with id, name, trigger description, and next_run_time
  start      Start the APScheduler background loop (blocking; Ctrl-C to stop)
  run-job    Invoke a single job callable by id immediately (for manual triggering)

Usage:
  python -m polytool research-scheduler status
  python -m polytool research-scheduler start
  python -m polytool research-scheduler start --dry-run
  python -m polytool research-scheduler run-job academic_ingest
  python -m polytool research-scheduler run-job weekly_digest --json
"""
```

**Subcommand: status**

Parse no extra args. Import `JOB_REGISTRY` from `packages.research.scheduling.scheduler`.
Print a table with columns: id, name, trigger_description. No APScheduler import needed for status (registry is always available). When `--json` flag present, output JSON list.

```
$ python -m polytool research-scheduler status
RIS Scheduler — Registered Jobs (8)

  ID                  Name                           Schedule
  ──────────────────────────────────────────────────────────────────
  academic_ingest     ArXiv academic ingestion       every 12h at 06:00 and 18:00
  reddit_polymarket   r/polymarket ingestion         every 6h at 00:00, 06:00, 12:00, 18:00
  ...
  weekly_digest       Weekly research digest         Sundays at 08:00

Note: Twitter/X ingestion is not scheduled (fetcher not yet implemented).
Run 'python -m polytool research-scheduler start' to launch the background loop.
```

**Subcommand: start**

Args: `--dry-run` (register jobs, print schedule, exit without starting).

When `--dry-run`: import scheduler module, print registered job ids and triggers, exit 0.

When not dry-run: call `start_research_scheduler()`. After start, print "Scheduler started. Press Ctrl-C to stop." Enter a `while True: time.sleep(60)` loop. On KeyboardInterrupt, call `scheduler.shutdown()`, print "Scheduler stopped.", exit 0.

Use `try: from apscheduler... except ImportError: print error and exit 1` at start of `_cmd_start()` to give a clear error if APScheduler is not installed.

**Subcommand: run-job**

Args: positional `job_id` (required), `--json` flag.

Import `run_job` from `packages.research.scheduling.scheduler`. Call `run_job(job_id)`. Return code 0 on success, 1 on unknown job id or exception. When `--json`: output `{"job_id": ..., "exit_code": ...}`.

Print `"Running job: {job_id} ({job_name})"` before invoking, then `"Done."` or `"Error: ..."` after.

**main(argv: list) -> int** — standard pattern, routes to subcommand handlers. Returns 0/1.
