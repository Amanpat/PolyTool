       - `load_freshness_config(config_path: Path | None = None) -> dict` -- loads from
         `config/freshness_decay.json` by default, returns parsed dict. Cache with
         `functools.lru_cache` on resolved path string.
       - `compute_freshness_modifier(source_family: str, published_at: datetime | None, config: dict | None = None) -> float`
         -- exponential decay: `max(floor, 2^(-age_months / half_life))`. Returns 1.0 if
         source_family has null half-life or published_at is None. Reads config via
         load_freshness_config if not provided. NEVER mutates stored records.

    3. Create `packages/polymarket/rag/knowledge_store.py`:
       - `KnowledgeStore(db_path: str | Path = "kb/rag/knowledge/knowledge.sqlite3")`
         -- accepts `:memory:` for tests. Creates parent dirs if needed. Calls
         `_ensure_schema()` on init.
       - Schema (4 tables):
         - `source_documents`: id TEXT PK, title TEXT, source_url TEXT, source_family TEXT,
           content_hash TEXT, chunk_count INTEGER, published_at TEXT, ingested_at TEXT,
           confidence_tier TEXT, metadata_json TEXT
         - `derived_claims`: id TEXT PK, claim_text TEXT NOT NULL, claim_type TEXT NOT NULL,
           confidence REAL NOT NULL, trust_tier TEXT NOT NULL, validation_status TEXT NOT NULL
           DEFAULT 'UNTESTED', lifecycle TEXT NOT NULL DEFAULT 'active',
           actor TEXT NOT NULL, created_at TEXT NOT NULL, updated_at TEXT NOT NULL,
           source_document_id TEXT REFERENCES source_documents(id),
           scope TEXT, tags TEXT, notes TEXT, superseded_by TEXT
         - `claim_evidence`: id INTEGER PK AUTOINCREMENT, claim_id TEXT NOT NULL
           REFERENCES derived_claims(id), source_document_id TEXT NOT NULL
           REFERENCES source_documents(id), excerpt TEXT, location TEXT,
           created_at TEXT NOT NULL
         - `claim_relations`: id INTEGER PK AUTOINCREMENT, source_claim_id TEXT NOT NULL
           REFERENCES derived_claims(id), target_claim_id TEXT NOT NULL
           REFERENCES derived_claims(id), relation_type TEXT NOT NULL
           CHECK(relation_type IN ('SUPPORTS','CONTRADICTS','SUPERSEDES','EXTENDS')),
           created_at TEXT NOT NULL
       - CRUD methods: `add_source_document(...)`, `add_claim(...)`,
         `add_evidence(...)`, `add_relation(...)`
       - Query methods:
         - `get_claim(claim_id) -> dict | None`
         - `query_claims(*, include_archived=False, include_superseded=False, apply_freshness=True) -> list[dict]`
           -- default excludes lifecycle='archived' and superseded claims.
           When apply_freshness=True, each result dict gets a `freshness_modifier` field
           computed via `compute_freshness_modifier`. Contradicted claims (any CONTRADICTS
           relation targeting them) get a penalty multiplier of 0.5 applied to their
           freshness_modifier. Results sorted by effective_score DESC (freshness_modifier *
           confidence * contradiction_penalty).
         - `get_provenance(claim_id) -> list[dict]` -- returns source docs linked via
           claim_evidence for the given claim.
         - `get_relations(claim_id, relation_type=None) -> list[dict]` -- returns
           relations where claim is source or target, optionally filtered by type.
       - Use `sqlite3` stdlib only (no ORM). Follow the pattern from `lexical.py`.
       - Use `hashlib.sha256` for deterministic IDs (follow existing `metadata.py` pattern).
       - Provider abstraction: include a `_llm_provider` attribute defaulting to None.
         Document in docstring: "LLM provider integration point for claim extraction.
         Cloud execution remains disabled by default pending authority sync between
         Roadmap v5.1 (Tier 1 free cloud APIs) and PLAN_OF_RECORD (no external LLM calls).
         See docs/features/FEATURE-ris-v1-data-foundation.md."
