    def evaluate_health(
        runs: list[RunRecord],

        audit_disagreement_rate: Optional[float] = None,
    ) -> list[HealthCheckResult]:
        Evaluate all checks and return one HealthCheckResult per check.

    Implement exactly these 6 checks (per RIS_06 health check table):

    1. "pipeline_failed" — RED when any run in `runs` has exit_status=="error"
       (checks the most recent run first; GREEN if no runs or all ok/partial)

    2. "no_new_docs_48h" — YELLOW when total accepted across window==0 AND len(runs)>0;
       GREEN when no runs (insufficient data); GREEN otherwise

    3. "accept_rate_low" — compute accept_rate = accepted/(accepted+rejected) across all
       runs in window; YELLOW when rate < 0.30 AND total > 5; GREEN otherwise or no data

    4. "accept_rate_high" — YELLOW when rate > 0.90 AND total > 10; GREEN otherwise

    5. "model_unavailable" — always GREEN in this pass (no provider event data fed in;
       deferred to when scheduler wires provider errors)

    6. "rejection_audit_disagreement" — YELLOW when audit_disagreement_rate is not None
       AND audit_disagreement_rate > 0.30; GREEN otherwise

    **packages/research/monitoring/alert_sink.py**

    AlertSink abstract base (Protocol or ABC with fire(result: HealthCheckResult) -> bool).

    LogSink(AlertSink): logs to logging.getLogger("ris.alerts"); fire() always returns True.

    WebhookSink(AlertSink): takes webhook_url param; fire() posts JSON payload via requests.post
    with 5s timeout; returns True on HTTP 200, False on any exception. Import requests inside
    fire() to keep it optional.

    def fire_alerts(
        results: list[HealthCheckResult],
        sink: AlertSink,
        *,
        min_level: str = "YELLOW",
    ) -> int:
        Fire sink.fire() for each result with status YELLOW or RED (not GREEN).
        Returns count of alerts fired.

    **tools/cli/research_health.py**

    def main(argv: list) -> int:
      argparse: --json (bool), --window-hours N (default 48), --run-log PATH (default
      artifacts/research/run_log.jsonl), --eval-artifacts PATH (default
      artifacts/research/eval_artifacts/eval_artifacts.jsonl).

      Steps:
      1. Load runs via list_runs(path, window_hours=window_hours). Gracefully returns []
         if file absent.
      2. Call evaluate_health(runs, window_hours=window_hours).
      3. Instantiate LogSink; call fire_alerts(results, sink) to emit log lines for issues.
      4. If --json: print JSON {"checks": [...], "summary": overall_status, "run_count": N}
         where overall_status = "RED" if any RED, "YELLOW" if any YELLOW, else "GREEN",
         and "no_data" when no runs.
      5. If not --json: print human-readable table:
         RIS Health Summary ({window_hours}h window, {run_count} runs)
         CHECK                    STATUS   MESSAGE
         pipeline_failed          GREEN    No pipeline errors
         ...
      6. Return 0 always (health output is informational; exit 1 would break cron).
