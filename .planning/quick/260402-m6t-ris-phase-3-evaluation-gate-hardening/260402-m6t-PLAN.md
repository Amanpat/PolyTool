<objective>
Add deterministic source-family feature extraction, near-duplicate detection, structured
scoring artifact persistence, and richer calibration analytics to the RIS evaluation gate.

Purpose: The eval gate currently relies entirely on LLM scoring after basic hard stops.
This plan adds structured signal extraction *before* the LLM, making the gate more
deterministic, observable, and tunable per source family -- without replacing LLM scoring.

Output:
- `feature_extraction.py` with per-family feature extractors
- `dedup.py` with content-hash and shingle-based near-duplicate detection
- `artifacts.py` for structured eval run persistence
- Enhanced `evaluator.py` pipeline: hard_stops -> dedup -> feature_extraction -> scoring -> artifact
- Richer calibration summaries (family-level, hard-stop causes, dedup rates)
- Comprehensive tests, feature doc, dev log
</objective>

    Feature extraction:
    - extract_features(doc: EvalDocument) returns FamilyFeatures dataclass
    - FamilyFeatures has: family (str), features (dict[str, Any]), confidence_signals (list[str])
    - Academic family: detects DOI pattern (10.NNNN/...), arXiv ID (YYMM.NNNNN), SSRN ID, methodology keywords (regression, p-value, sample size, dataset, experiment, methodology, control group), publication metadata presence (author != "unknown", publish_date not None)
    - GitHub family: detects stars/forks from metadata dict if present, README mention in body, license keyword in body, commit-recency field in metadata if present
    - Blog/news family: detects byline pattern (by [Name] or author attribution), date presence, structural markers (h1/h2/h3 markdown headings count, paragraph count, blockquote presence)
    - Forum/social family: detects screenshot/image references, data/chart mentions, thread-length proxies (reply count in metadata), specificity markers (numbers, percentages, specific dates)
    - Manual/other family: returns minimal features (body_length, has_url, word_count)
    - All extraction is pure text/regex, no network calls

    Dedup:
    - compute_content_hash(body: str) returns SHA256 hex string of normalized (lowered, whitespace-collapsed) body
    - compute_shingles(body: str, shingle_size: int = 5) returns frozenset of word n-gram tuples
    - jaccard_similarity(a: frozenset, b: frozenset) returns float 0.0..1.0
    - check_near_duplicate(doc: EvalDocument, existing_hashes: set[str], existing_shingles: list[tuple[str, frozenset]]) returns NearDuplicateResult(is_duplicate: bool, duplicate_type: Optional[str], matched_doc_id: Optional[str], similarity: float)
    - Exact duplicate: content_hash in existing_hashes
    - Near-duplicate: jaccard_similarity > 0.85 threshold against any existing shingle set
    - Empty body returns NearDuplicateResult(is_duplicate=False) without error

    Artifacts:
    - EvalArtifact dataclass: doc_id, timestamp, gate, hard_stop_result (dict|None), near_duplicate_result (dict|None), family_features (dict), scores (dict|None), source_family (str), source_type (str)
    - persist_eval_artifact(artifact: EvalArtifact, artifacts_dir: Path) writes one JSONL line to artifacts_dir/eval_artifacts.jsonl
    - load_eval_artifacts(artifacts_dir: Path) returns list[dict] from JSONL
    - Creates artifacts_dir if not exists
    - Handles empty/missing file gracefully (returns [])

    2. Create `packages/research/evaluation/dedup.py`:
       - `compute_content_hash(body: str) -> str`: normalize body (lower, strip, collapse whitespace via `" ".join(body.lower().split())`), return `hashlib.sha256(normalized.encode("utf-8")).hexdigest()`
       - `compute_shingles(body: str, shingle_size: int = 5) -> frozenset`: split normalized body into words, generate n-gram tuples of size `shingle_size`, return as frozenset
       - `jaccard_similarity(a: frozenset, b: frozenset) -> float`: `len(a & b) / len(a | b)` if union non-empty else 0.0
       - `NearDuplicateResult` dataclass: `is_duplicate: bool`, `duplicate_type: Optional[str]` ("exact" | "near" | None), `matched_doc_id: Optional[str]`, `similarity: float`
       - `check_near_duplicate(doc, existing_hashes: set[str], existing_shingles: list[tuple[str, frozenset]], threshold: float = 0.85) -> NearDuplicateResult`:
         - If body is empty/None, return not-duplicate
         - Compute content hash. If in existing_hashes, return exact duplicate
         - Compute shingles. For each (doc_id, shingle_set) in existing_shingles, compute jaccard. If any > threshold, return near-duplicate with that doc_id and similarity
         - Otherwise return not-duplicate
