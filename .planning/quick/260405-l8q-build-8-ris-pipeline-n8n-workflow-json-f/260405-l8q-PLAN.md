     const output = $input.first().json.stdout || '';

         hasRed,
         exitCode,
         output: output.substring(0, 1500),

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| n8n -> host Docker socket | n8n executes docker exec commands on the host |
| Webhook -> n8n | External HTTP POST triggers command execution |
| n8n -> Discord | Outbound webhook with workflow output data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-quick-01 | T (Tampering) | Webhook endpoint | accept | Webhook path contains n8n-generated token; same risk profile as existing v1 ris_manual_acquire.json; internal-only network |
| T-quick-02 | I (Information Disclosure) | Discord embeds | mitigate | Truncate stdout/stderr in Discord embeds to 1500/500 chars to avoid leaking full command output; no secrets in research-health or research-scheduler output |
| T-quick-03 | E (Elevation of Privilege) | docker exec | accept | Same docker-beside-docker pattern as v1 (quick-260404-t5l); n8n container already has Docker socket access per ADR-0013; scoped to polytool-polytool-1 container only |
| T-quick-04 | S (Spoofing) | Webhook input | mitigate | Execute Command uses expression interpolation of url/source_family from body; polytool research-acquire CLI validates source_family server-side; URL passed as quoted argument |
</threat_model>
