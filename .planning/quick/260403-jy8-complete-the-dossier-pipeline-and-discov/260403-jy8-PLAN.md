Dossier artifact structure (per run directory):
- dossier.json: keys = [anchors, coverage, detectors, distributions, header, liquidity_summary, pnl_summary, positions, schema_version]
  - header: {export_id, generated_at, max_trades, proxy_wallet, user_input, window_days, window_end, window_start}
  - detectors.latest: list of {bucket_start, detector, label, score}
  - pnl_summary: {latest_bucket, pricing_confidence, pricing_snapshot_ratio, trend_30d}
- memo.md: LLM research packet with executive summary, key observations, hypotheses table, evidence anchors
- hypothesis_candidates.json: {candidates: [{clv_variant_used, denominators, falsification_plan, metrics, ...}], generated_at, run_id, user_slug, wallet}
- segment_analysis.json: {generated_at, run_id, segment_analysis, user_slug, wallet}
- run_manifest.json: {argv, command_name, diagnostics, duration_seconds, ...}

Dossier directory layout:
  artifacts/dossiers/users/{user_slug}/{wallet}/{date}/{run_id}/
  Example: artifacts/dossiers/users/gabagool22/0x640a.../2026-03-29/47c5ac46-.../

    **Create `packages/research/integration/dossier_extractor.py`** — the core module:

    1. `_parse_dossier_json(path) -> dict`: Load dossier.json. Extract:
       - From header: user_input, proxy_wallet, window info, export_id
       - From detectors.latest: strategy classification labels (COMPLETE_SET_ARBISH, DCA_LADDERING, MARKET_SELECTION_BIAS, HOLDING_STYLE)
       - From pnl_summary: pricing_confidence, trend summary
       Return a structured dict with these extracted fields.

    2. `_parse_memo(path) -> str`: Read memo.md. Strip TODO placeholders.
       Return the usable body text. If mostly TODOs, return empty string.

    3. `_parse_hypothesis_candidates(path) -> list[dict]`: Load hypothesis_candidates.json.
       Extract top candidates with metrics (clv_variant_used, key metrics like avg_clv_pct, beat_close_rate, count).
       Return list of candidate summary dicts.

    4. `_build_finding_documents(dossier_dir, parsed_dossier, parsed_memo, parsed_candidates) -> list[dict]`:
       Combine parsed outputs into a list of document dicts ready for IngestPipeline.
       Each document dict has: title, body, source_url (file:// URI to dossier_dir),
       source_family ("dossier_report"), author (user_slug from header or dir name),
       publish_date (generated_at from header), metadata (wallet, run_id, dossier_path,
       detector_labels, strategy_tracks).
       Produce 1-3 documents per dossier run:
       - "Dossier Detectors: {user_slug}" — strategy classification + detector labels
       - "Dossier Hypothesis Candidates: {user_slug}" — if candidates exist, summarize top metrics
       - "Dossier Memo: {user_slug}" — if memo has non-TODO content

    5. `extract_dossier_findings(dossier_dir: str|Path) -> list[dict]`:
       Top-level function. Reads all files from a single run directory.
       Calls _parse_* helpers, then _build_finding_documents. Returns list of document dicts.

    6. `batch_extract_dossiers(base_dir: str|Path) -> list[dict]`:
       Walk artifacts/dossiers/users/{slug}/{wallet}/{date}/{run_id}/ tree.
       For each run_id dir that contains dossier.json, call extract_dossier_findings.
       Return flat list of all finding dicts across all dossiers.

    7. `ingest_dossier_findings(findings: list[dict], store: KnowledgeStore, post_extract_claims: bool = False) -> list[IngestResult]`:
       For each finding dict, call IngestPipeline(store).ingest(finding body text,
       source_type="dossier", title=finding.title, post_ingest_extract=post_extract_claims, ...).
       Use content_hash for dedup (SHA-256 of body). Return list of IngestResults.
