**tools/cli/research_stats.py**

Implements `research-stats` CLI with two subcommands: `summary` and `export`.

Module docstring:
```python
"""CLI entrypoint for RIS operator stats and metrics export.

Subcommands:
  summary  Print a human-readable metrics snapshot to stdout.
  export   Write metrics_snapshot.json to artifacts/research/ (local-first Grafana file).

Usage:
  python -m polytool research-stats summary
  python -m polytool research-stats summary --json
  python -m polytool research-stats export
  python -m polytool research-stats export --out artifacts/research/metrics_snapshot.json
"""
```

`summary` subcommand:
- Call `collect_ris_metrics()` with all defaults.
- If `--json`: print `json.dumps(snapshot.to_dict(), indent=2)` to stdout.
- Otherwise: print `format_metrics_summary(snapshot)` to stdout.
- Exit 0.

`export` subcommand:
- `--out PATH` (default: `artifacts/research/metrics_snapshot.json`)
- Call `collect_ris_metrics()` with all defaults.
- Write `json.dumps(snapshot.to_dict(), indent=2)` to the output path. Create parent dirs.
- Print `Metrics exported to: {path}` to stdout.
- Exit 0.

Additional flags on both subcommands:
- `--db PATH`: override KS db path
- `--eval-artifacts-dir PATH`: override eval artifacts dir
- `--precheck-ledger PATH`: override precheck ledger path
- `--report-dir PATH`: override report dir
- `--acquisition-review-dir PATH`: override acquisition review dir

`main(argv: list) -> int` follows the same pattern as `research_report.py`:
- `argparse.ArgumentParser(prog="research-stats", ...)`
- subparsers with `dest="subcommand"`
- Route to `_cmd_summary(args)` or `_cmd_export(args)`
- On unknown subcommand: print help and return 1
