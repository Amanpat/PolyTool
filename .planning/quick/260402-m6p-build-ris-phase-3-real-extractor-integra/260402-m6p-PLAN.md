<objective>
Build RIS Phase 3: real extractor integration and corpus backfill.

Replace stub extractors with real implementations, add a structure-aware Markdown
extractor that materially improves ingestion quality over plain text, expand the
benchmark harness with quality proxy metrics, and wire a reseed workflow that can
reingest the full docs/reference/ corpus with improved extraction without breaking
document identity or provenance.

Purpose: The RIS knowledge store currently treats all Markdown as flat text. The
entire docs/reference/ corpus is Markdown (no PDF/DOCX files exist yet), so the
highest-impact improvement is a structure-aware Markdown extractor. PDF and DOCX
extractors are added as optional-dep real implementations (pdfplumber, python-docx)
with graceful ImportError fallback to the stubs, so when those file types arrive
the path is ready.

Output:
- StructuredMarkdownExtractor in extractors.py (section-aware, preserves headers/tables/code)
- Real PDFExtractor (pdfplumber, optional dep) and DocxExtractor (python-docx, optional dep)
- Enhanced benchmark harness with quality proxy metrics
- Expanded seed manifest covering full docs/reference/ corpus
- Reseed capability on research-seed CLI (--reseed flag)
- Tests, feature doc, dev log
</objective>
