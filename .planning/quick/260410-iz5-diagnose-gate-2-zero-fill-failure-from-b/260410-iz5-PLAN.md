  <action>
Create `tools/gates/diagnose_zero_fill.py` that replays a single tape through the
full SimTrader pipeline with tick-by-tick diagnostic instrumentation. The script must
answer four diagnostic questions at every tick:

**Q1 — Book initialization**: Is L2Book._initialized True? How many bid/ask levels exist?
Record per-tick: `{seq, book_initialized, n_bids, n_asks, best_bid, best_ask}`.

**Q2 — Quote marketability**: What bid/ask does MarketMakerV1._compute_quotes() produce,
and do the quotes cross the book BBO? Record per-tick (when strategy quotes):
`{seq, strat_bid, strat_ask, book_best_bid, book_best_ask, bid_crosses_ask, ask_crosses_bid}`.
"Crosses" means the strategy's bid >= book best_ask (would fill as BUY) or strategy's
ask <= book best_bid (would fill as SELL).

**Q3 — Reservation/inventory blocking**: Does the "Insufficient position to reserve SELL
order" warning fire? Capture the reservation check logic. Record per-tick if SELL intent
is generated but rejected at reservation stage.

**Q4 — Fill rejection reasons**: What `reject_reason` does FillRecord carry for each
rejected fill? Aggregate counts: `{book_not_initialized: N, no_competitive_levels: N, ...}`.

Implementation approach:
- Load events from tape `events.jsonl` (same as StrategyRunner does).
- Create L2Book, SimBroker, and MarketMakerV1 instances directly (do NOT subclass StrategyRunner;
  reproduce its loop manually so you can inspect intermediate state).
- At each event:
  1. Record pre-apply book state (initialized? levels?)
  2. Apply event to book
  3. Record post-apply book state
  4. Call strategy.on_event() to get OrderIntents
  5. For each OrderIntent, record: asset_id, side, limit_price, size
  6. Submit orders to broker
  7. Call broker.step() and capture all FillRecords
  8. For rejected fills, record reject_reason from FillRecord
  9. For "Insufficient position to reserve" — hook logging to capture this
- At end, emit a JSON diagnostic report to stdout and optionally to --out file:
  ```json
  {
    "tape_dir": "...",
    "tape_tier": "silver|gold|unknown",
    "total_events": N,
    "book_affecting_events": N,
    "book_ever_initialized": true|false,
    "first_book_init_seq": null|N,
    "event_type_counts": {"price_change": N, "book": N, "last_trade_price": N},
    "quote_ticks": N,
    "quote_samples": [first 5 and last 5 quote ticks with full detail],
    "fill_attempts": N,
    "fill_successes": N,
    "fill_rejection_counts": {"book_not_initialized": N, "no_competitive_levels": N},
    "reservation_blocks": {"sell_insufficient_position": N},
    "order_intents_by_side": {"BUY": N, "SELL": N},
    "verdict": "BOOK_NEVER_INITIALIZED|NO_COMPETITIVE_LEVELS|RESERVATION_BLOCKED|QUOTES_TOO_WIDE|FILLS_OK|UNKNOWN",
    "verdict_evidence": "human-readable explanation"
  }
  ```

Verdict logic (deterministic):
- If `book_ever_initialized == false` -> "BOOK_NEVER_INITIALIZED"
- Else if all rejections are `no_competitive_levels` -> "NO_COMPETITIVE_LEVELS"
- Else if `reservation_blocks.sell_insufficient_position > 0` and that accounts for >50% of SELL attempts -> "RESERVATION_BLOCKED"
- Else if quotes never cross BBO -> "QUOTES_TOO_WIDE"
- Else if fills > 0 -> "FILLS_OK"
- Else -> "UNKNOWN"

CLI interface:
```
python tools/gates/diagnose_zero_fill.py --tape-dir <PATH> --asset-id <TOKEN_ID> [--out <PATH>] [--verbose]
```
- `--tape-dir`: path to a tape directory containing events.jsonl
- `--asset-id`: YES token ID
