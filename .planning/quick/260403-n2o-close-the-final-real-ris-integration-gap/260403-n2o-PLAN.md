<objective>
Close the final functional gap in the RIS dossier integration: make wallet-scan
--extract-dossier produce retrieval-visible KnowledgeStore content (derived_claims),
not just ingested source_documents.

Purpose: Currently, ingest_dossier_findings() stores dossier findings as source_documents
but does NOT run claim extraction (post_extract_claims defaults to False). The hybrid
retrieval path (query_knowledge_store_for_rrf) queries derived_claims only. So dossier
findings are stored but invisible to retrieval. This is the gap Codex flagged.

Output: The shipped --extract-dossier flow yields queryable KS-backed results through
the hybrid retrieval path, with truthful provenance and tests proving it.
</objective>
