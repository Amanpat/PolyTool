Diagnose and fix the n8n instance-level MCP connection for Claude Code.

Purpose: The n8n-instance-mcp entry in .mcp.json uses `${N8N_BASE_URL}` and
`${N8N_MCP_TOKEN}` env-var expansion syntax, but Claude Code does NOT expand
env vars in HTTP-type MCP server URLs/headers. The literal template strings are

   URL shows literal `${N8N_BASE_URL}/mcp-server/http` in `claude mcp list`

   JSON-RPC MCP response when given correct JWT + Accept headers)
3. `.env.example` line 149 contains a real JWT token (HS256-signed, iss=n8n,
   aud=mcp-server-api) -- must be replaced with a placeholder
4. Docs (RIS_OPERATOR_GUIDE.md, infra/n8n/README.md, dev logs) all claim
   "Enterprise only" based on earlier flawed probes that sent GET requests
   without proper Accept/Content-Type headers

Output: Fixed .mcp.json, cleaned .env.example, corrected docs, dev log.
