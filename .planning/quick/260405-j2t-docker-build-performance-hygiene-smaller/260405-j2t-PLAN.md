**1B. Fix root `Dockerfile` cache layer ordering and add BuildKit cache mounts:**

Current problem: `COPY . .` comes before `pip install ".[all,ris]"`. This means ANY source file edit invalidates the pip install layer. The `COPY pyproject.toml ./` + `pip install py-clob-client` partial pre-install helps somewhat, but the main `.[all,ris]` install still gets busted on every edit.

Rewrite root Dockerfile to:

```dockerfile
# syntax=docker/dockerfile:1
FROM python:3.11-slim

ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    PIP_DISABLE_PIP_VERSION_CHECK=1

RUN --mount=type=cache,target=/var/cache/apt,sharing=locked \
    --mount=type=cache,target=/var/lib/apt/lists,sharing=locked \
    apt-get update \
    && apt-get install -y --no-install-recommends gcc libffi-dev curl

RUN groupadd -r polytool && useradd -r -g polytool -m polytool

WORKDIR /app

# Layer 1: dependency manifests only — changes rarely
COPY pyproject.toml ./

# Layer 2: install ALL dependencies (cached unless pyproject.toml changes)
RUN --mount=type=cache,target=/root/.cache/pip \
    pip install --upgrade pip \
    && pip install ".[all,ris]"

# Layer 3: copy source code (changes often, but deps are cached above)
COPY polytool/ ./polytool/
COPY packages/ ./packages/
COPY tools/ ./tools/
COPY services/ ./services/

# Re-install in editable-like mode so package metadata is correct
# (setuptools needs pyproject.toml + source to resolve entry points)
RUN --mount=type=cache,target=/root/.cache/pip \
    pip install --no-deps ".[all,ris]"

RUN chown -R polytool:polytool /app

USER polytool
```

Key changes:
- `# syntax=docker/dockerfile:1` enables BuildKit syntax
- `--mount=type=cache` for apt and pip — no re-download on rebuilds
- Removed `--no-cache-dir` from pip (let BuildKit cache handle it)
- Selective COPY of only source directories instead of `COPY . .`
- Two-phase pip install: first with deps (cached), then `--no-deps` after source copy for metadata
- apt lists cleanup removed (cache mount handles it)

**1C. Add BuildKit cache mounts to `services/api/Dockerfile`:**

```dockerfile
# syntax=docker/dockerfile:1
FROM python:3.11-slim

RUN --mount=type=cache,target=/var/cache/apt,sharing=locked \
    --mount=type=cache,target=/var/lib/apt/lists,sharing=locked \
    apt-get update \
    && apt-get install -y --no-install-recommends curl

WORKDIR /app

# Install dependencies (cached unless requirements.txt changes)
COPY services/api/requirements.txt /app/requirements.txt
RUN --mount=type=cache,target=/root/.cache/pip \
    pip install -r requirements.txt

# Copy packages
COPY packages /app/packages

# Copy service code
COPY services/api /app/services/api

# Set Python path
ENV PYTHONPATH=/app/packages:/app

WORKDIR /app/services/api

EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

Minimal change — just adds BuildKit syntax line and cache mounts. Layer ordering was already good (requirements.txt first, then source).
