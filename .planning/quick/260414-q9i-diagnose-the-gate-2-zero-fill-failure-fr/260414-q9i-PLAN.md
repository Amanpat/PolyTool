<hypothesis>
## Pre-Analysis: Three Candidate Root Causes

Based on code review before any diagnostic runs:

**H1 (MOST LIKELY): Silver tapes lack order book depth beyond BBO.**
Silver tapes are reconstructed from `price_2min` guide data. The L2Book is populated
via `apply()` / `apply_single_delta()` from tape events. If Silver tape events only
contain BBO-level data (best bid/ask prices with their sizes), then:
- The MM posts a BUY bid BELOW the current best bid (passive quoting).
- For that BUY to fill, the book's ask side must contain a level AT or BELOW the bid price.
- But if the book only has ONE ask level (the BBO), and the MM's bid is below the BBO ask,
  the fill engine finds no competitive ask levels -> "no_competitive_levels" -> 0 fills.
- The book must MOVE DOWN (ask drops to/below the bid) for a fill. With 2-min granularity
  and only ~30 events, there may never be enough movement.

**H2: Strategy quotes too wide due to resolution guard.**
All 9 qualifying tapes are `near_resolution` bucket. If mid > 0.90 (or < 0.10),
the resolution guard multiplies the logit spread by 2.5x. Combined with the
spread_multiplier sweep (0.50x to 3.00x), total spread could be 0.50*2.5 = 1.25x
to 3.00*2.5 = 7.5x base. At extreme mids (p~0.95), even a 1.25x logit spread
may produce quotes so far from the BBO that no 2-min price movement crosses them.

**H3: Chicken-and-egg inventory issue.**
The SELL side is always blocked (no inventory). If BUY fills never happen (H1/H2),
then SELL orders never have inventory to reserve against. This is a CONSEQUENCE, not
a cause. The portfolio reservation warning is informational only; orders still reach
the broker.

## Diagnostic Strategy

The script must answer: at each tick where the MM submits a BUY order, what are:
1. The BUY limit_price (the MM's bid)
2. The book's ask levels at that moment (the available counterparty liquidity)
3. Whether ANY ask level is at/below the BUY limit_price
4. If not, what is the gap (best_ask - bid) in cents

If the answer to (3) is NEVER yes across all ticks on all tapes, the root cause is
confirmed: Silver tapes do not contain ask levels that overlap with the MM's quotes.
The next question is whether this is because (a) the book has depth but the quotes
are too wide, or (b) the book only has BBO and no depth.
</hypothesis>
