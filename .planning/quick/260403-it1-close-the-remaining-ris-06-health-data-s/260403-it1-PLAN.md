<objective>
Wire the manual operator CLI commands (research-ingest, research-acquire) into the RIS
run log so that manual runs produce real RunRecords visible to research-health. Fix
health check output to honestly label deferred/stubbed checks instead of showing
misleading GREEN. Add integration tests proving the ingest->run_log->health pipeline.

Purpose: The RIS health surface currently shows "no_data" after manual operator runs
because research-ingest and research-acquire do not call append_run(). This makes
research-health useless for the most common operator workflow. Fixing this closes the
last data-surface gap in RIS_06.

Output: Updated CLI files, improved health labeling, integration tests, dev log.
</objective>
