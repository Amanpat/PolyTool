```python
PAPER_SOAK_VERDICT_JSON = "paper_soak_verdict.json"
```

**1b. Write standalone verdict.json in `generate_crypto_pair_paper_report`:**

After computing the report dict and writing summary artifacts, write a lightweight
`paper_soak_verdict.json` containing only:
```json
{
  "schema_version": "crypto_pair_verdict_v0",
  "run_id": "<from report>",
  "generated_at": "<from report>",
  "decision": "promote" | "rerun" | "reject",
  "verdict": "<human-readable verdict string>",
  "rubric_pass": true | false,
  "safety_violation_count": 0,
  "decision_reasons": ["..."],
  "net_pnl_usdc": 1.56,
  "soak_duration_hours": 24.0
}
```

This is a SUBSET of the full report -- the minimum an operator needs to triage.
Write it to `loaded_run.run_dir / PAPER_SOAK_VERDICT_JSON`.

Populate `result.verdict_path` in the returned `CryptoPairReportResult`.

**1c. Enrich `build_paper_soak_summary` with operational context:**

Add a new top-level key `"operational_context"` to the report dict returned by
`build_paper_soak_summary`. Extract the following from the already-loaded artifacts:

- `cycles_completed`: from `manifest["runner_result"]["cycles_completed"]` if present,
  else count `cycle_completed` events in `runtime_events`, else `None`.
- `symbols_included`: sorted unique list of symbols from the market-to-symbol index
  (already built as `market_to_symbol`). Example: `["BTC", "ETH", "SOL"]`.
- `markets_observed_count`: already in metrics as `opportunities_observed`, reference it.
- `markets_by_symbol`: dict mapping each symbol to the count of unique market_ids
  observed for that symbol. Build from the `market_to_symbol` index.

This requires no new artifact reads -- `market_to_symbol` is already computed,
manifest is already loaded.

**1d. Surface operational context in markdown rendering:**

In `render_paper_soak_summary_markdown`, add the operational context fields to the
Key Metrics table. Add these rows after `soak_duration_hours`:
- `cycles_completed`
- `symbols_included` (join list with ", ")
- `markets_by_symbol` (render as "BTC=4, ETH=4, SOL=4")

**1e. Update `build_report_artifact_paths`:**

Add `"verdict_json": str(result.verdict_path)` to the returned dict.

Do NOT touch `tools/cli/crypto_pair_run.py` or `tools/cli/crypto_pair_report.py` --
they already print artifact paths from `build_report_artifact_paths` and the existing
auto-report integration will surface the verdict path automatically.
