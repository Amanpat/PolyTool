### claim_extractor.py (current behavior)

- Public function: `extract_claims_from_document(store, doc_id) -> list[str]`
- It loads the doc body via `_get_document_body(store, doc)` (reads `metadata_json.body` or file:// URL)
- It then calls `chunk_text(body)` and iterates chunks, calling `_extract_assertive_sentences(chunk.text)` on each
- `_extract_assertive_sentences` does best-effort heading/table/code stripping, BUT the chunker flattens the whole body (including YAML frontmatter) into space-joined words. Lines like `title: "Polymarket Fee Structure (April 2026)"`, `freshness_tier: CURRENT`, `confidence_tier: PRACTITIONER` currently flow through as "assertive sentences" once they hit the chunk boundary that stitches multiple frontmatter lines into a period-terminated run.
- Constant to reuse: `EXTRACTOR_ID = "heuristic_v1"`. Bump to `"heuristic_v2_nofrontmatter"` so deterministic claim IDs regenerate cleanly (existing claims remain under v1; new run creates v2 rows without collision).

### YAML frontmatter shape in the 7 docs

Every doc in `docs/external_knowledge/*.md` starts with:
```
---
title: "..."
freshness_tier: CURRENT
confidence_tier: PRACTITIONER|COMMUNITY
validation_status: UNTESTED
source_family: external_knowledge
source_quality_caution: >
  ... (multi-line folded block possible)
---
```
Stripping rule: if `body.lstrip().startswith("---\n")` (or `---\r\n`), find the SECOND occurrence of a line that is exactly `---` (with optional trailing whitespace) and remove everything up to and including that closing fence + following newline. If no closing `---` is found within the first 200 lines, leave the body unchanged (defensive: avoid nuking legitimate content with stray leading `---`).

### KnowledgeStore claim path

- `store.add_claim(..., created_at=det_created_at)` is idempotent via `INSERT OR IGNORE`
- Claim IDs are content-addressed: `_sha256_id("claim", claim_text, actor, created_at)`
- Because `actor=EXTRACTOR_ID`, changing EXTRACTOR_ID to `heuristic_v2_nofrontmatter` yields fresh claim IDs even for sentences that happen to match old ones -- this is the intended behavior for a re-extraction pass.
