## RISK NOTES

1. **Vacuous health GREEN is a truth-drift risk**: The monitoring layer (evaluate_health,
   research-health CLI) works correctly against real RunRecord data. However, because
   run_job() does not yet call append_run, any real scheduler invocation produces no log
   entries. An operator running "research-health" after a real scheduler run will see
   "GREEN" — not because the pipeline is healthy, but because there is no data at all.
   This is a silent false positive. Fix: 260403-2ow.

   returns GREEN (documented in health_checks.py). It is waiting for ProviderEvent data
   to be wired. This is expected and documented, not a defect.

   audit runner data. If no audit runner has produced data, it may return vacuously GREEN

   does not yet note that APScheduler is optional ([ris] extras group). An operator reading
